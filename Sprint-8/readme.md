# 🧭 Guide to Sprint 8 — Control Flow & Logic

> [!TIP]
> You should always do the prep work _before_ attempting the coursework.

## Overview

In this sprint, you'll master **control flow** through a real-world scenario: building a **ticket booking and pricing system**. You'll learn conditional logic, switch statements, ternary operators, and complex decision-making.

**Real-world application:** Ticketmaster, Eventbrite, airline booking systems all use complex pricing rules based on multiple factors.

## Learning Objectives

By the end of this sprint, you will be able to:

- ✅ Use `if`, `else if`, and `else` statements effectively
- ✅ Write switch statements for multiple conditions
- ✅ Use ternary operators for concise conditions
- ✅ Combine logical operators (`&&`, `||`, `!`)
- ✅ Handle complex conditional logic
- ✅ Write guard clauses and early returns
- ✅ Implement dynamic pricing rules

## Structure

### 1. Key Exercises (`1-key-exercises/`)
1. **Age-based pricing** - Simple if/else
2. **Peak pricing** - Time-based rules
3. **Switch statements** - Day of week logic
4. **Ternary operators** - Concise conditions
5. **Complex conditions** - Multiple factors

### 2. Mandatory Implement (`2-mandatory-implement/`)
1. `calculateTicketPrice()` - Dynamic pricing
2. `checkAvailability()` - Seat availability logic
3. `applyDiscount()` - Discount rules
4. `validateBooking()` - Booking validation

### 3. Stretch Goals (`3-stretch/`)
- Complete reservation system
- Implement group bookings
- Add promo code system
