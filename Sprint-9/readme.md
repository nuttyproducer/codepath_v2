# 🧭 Guide to Sprint 9 — Error Handling & Debugging

> [!TIP]
> You should always do the prep work _before_ attempting the coursework.

## Overview

In this sprint, you'll master **error handling** through a real-world scenario: building a **banking transaction system**. You'll learn how to handle errors gracefully, validate data, and write defensive code.

**Real-world application:** Revolut, N26, and online banking apps need robust error handling to protect user money and provide clear feedback when things go wrong.

## Learning Objectives

By the end of this sprint, you will be able to:

- ✅ Use `try...catch` blocks to handle errors
- ✅ Throw custom errors with meaningful messages
- ✅ Create custom error classes
- ✅ Validate inputs before processing
- ✅ Write defensive code
- ✅ Debug with console methods
- ✅ Use browser DevTools effectively
- ✅ Handle asynchronous errors

## Structure

### 1. Key Exercises (`1-key-exercises/`)
1. **Try-catch basics** - Error handling fundamentals
2. **Throw errors** - Creating error messages
3. **Custom errors** - Error classes
4. **Input validation** - Defensive coding
5. **Debugging techniques** - Finding bugs

### 2. Mandatory Implement (`2-mandatory-implement/`)
1. `validateTransaction()` - Check transaction validity
2. `processPayment()` - Handle payment errors
3. `handleInsufficientFunds()` - Custom error handling
4. `logTransaction()` - Error logging

### 3. Stretch Goals (`3-stretch/`)
- Complete banking system with error handling
- Transaction rollback on failure
- Comprehensive error logging system
